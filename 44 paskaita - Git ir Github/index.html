Git ir Github

Git'as - kodo versijavimo programa.
GitHub'as - versijuoto kodo talpykla internete.

Git komandos

  • init - sukurti git repozitoriją dabartiniame aplankale
  • status - pateikia git repozitorijos dabartinę informaciją
  • add - prideda nurodytus failus prie commit'inimui paruoštų failų (stage'ina)
  • rm --cacheed || restore --staged - išima nurodytus failus ir stage'inimo zonos(unstage'ina)
  • commit -m "žinutė apie commit'ą" - sukuria kodo versiją iš stage'intų failų ir prideda žinutę apie pakitimus (žinutė turėtų būti trumpa ir aiški)
  • restore - sugrąžina nurodytą/us failą/us į pradinę būseną (jeigu tie failai nebuvo add'inti į stage'ingo zoną)
  • push - nusiunčia lokalios repozitorijos commit'us į išorinę repozitoriją
  • pull - atsiunčia išorinės repozitorijos commit'us į lokalią repozitoriją
  • log - išveda commit'ų sąrašą
  • clone - nuklonuoja nurodytą projektą į dabartinį aplankalą
  • checkout commitID|branchName - pereinama į nurodytą kodo versiją arba kitą šaką.
  • branch naujosSakosPavadinimas - sukuriama nauja šaka

Branch protection rules

Vienas svarbiausių šakų apsaugos taisyklių yra "Pull Request" (bandymas vienos šakos commit'us sukelti į kitą šaką(merge)).
Pull Request taisyklės viduje dar reikėtų suteikti "Require approvals" nustatymą, kuris reikalauja, kad nurodytas kiekis kitų projekto dalyvių patvirtintų tą Pull Request'ą, norint, kad šakos būtų apjungtos.

Eiliškumas Pull Request

Apsiėmiau daryti kažkokią užduotį. Kokie mano tolimesni veiksmai?

  1. Susikurti naują šaką - git branch naujaSaka
  2. Nueiti į susikurtą šaką - git checkout naujaSaka
  3. Atlieki užduotį, kurią apsiėmei daryti. Su vienu arba keliais commit'ais
    1. Add'ini pakeitimus - git add .
    2. Commit'ini pakitimus - git commit -m "žinutė apie pakitimus"
  4. Nueiti į main/master šaką - git checkout main/maister
  5. Atnaujinti main/master šaką - git pull
  6. Grįžti į savo užduoties šaką - git checkout naujaSaka
  7. Sujungti savo šaką su main/master šaka ir išspręsti galimai atsiradusius konfliktus - git merge main (sujunginėti būnant savo užduoties šakoje)
  8. Nusiunti savo darbinės šakos pakitimus į gitHub'ą - git push
  9. GitHub'e sukurti Pull Request'ą
  10. Kiti komandos nariai arba patvirtina, arba atmeta Pull Request'ą
  11. Jeigu Pull Request'as sulaukė pakankamai patvirtinimų - vykdyti merge'ą
